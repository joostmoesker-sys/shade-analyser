Generic PV battery shadow tool implementation progress
======================================================

Status date: 2026-05-06

Implemented before this slice
-----------------------------
- Created a separate dependency-free generic prototype under generic/.
- Added workflow tabs for location, objects, PV arrays, wiring, inverters, battery/load, simulation, and results.
- Added a versioned project model with scenarios, location, panel types, scene objects, arrays, inverter/MPPT data, battery/load data, tariffs, validation, preview simulation, localStorage, JSON import/export, and README discovery link.
- Added SVG canvas rendering for arrays and basic shadow objects.
- Added fast yearly preview simulation with monthly PV/load aggregation and battery self-consumption estimate.

Implemented in this slice
-------------------------
- Added weather dataset metadata to each scenario, including source, year, profile, and completeness placeholders.
- Added editable tariff settings to support the PRD's economic simulation direction.
- Added more PRD scene object types: chimney, dormer, and free object placeholders.
- Added a reusable wiring module with panel-grid generation and configurable wiring summaries.
- Added wiring modes for row strings in parallel, column strings in parallel, and one full-array series string.
- Updated technical validation and preview simulation to use wiring summaries instead of a hard-coded row-string assumption.
- Added panel type creation/editing so users can enter custom panel specifications.
- Added UI controls for extra object types, wiring mode selection, weather profile/year, tariffs, array loss details, and CSV result export.
- Added scenario renaming, run-all-scenarios preview execution, and a results comparison table for simulated scenarios.
- Replaced the empty local-grid canvas with a vanilla OpenStreetMap slippy map under the existing SVG project overlay.
- Added address/postcode search via OpenStreetMap Nominatim in the Locatie workflow.
- Added map click location selection that updates project name, latitude and longitude fields.
- Added basic map pan/zoom controls while preserving existing PV-array and shadow-object overlay rendering for phase-2 work.

Next planned implementation items
---------------------------------
- Make phase-2 object placement use map coordinates instead of local overlay coordinates.
- Implement interactive panel/string selection on top of the map canvas instead of mode-based wiring only.
- Add hourly weather import/cache model and replace monthly preview factors with hourly simulation inputs.
- Add per-panel shade/production heatmap output.
- Add battery dispatch strategies for grid pre-charge and dynamic tariffs.
